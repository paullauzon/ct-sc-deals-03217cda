## Open issue — AI-tier enrichment

**`firm_aum` / `deal_type` / `transaction_type` are still 0%.** Two attempts to run `bulk-enrich-sourceco` on 150–200 leads:
- **Sync mode**: edge proxy 150s timeout kills the function before completion (logs show `enrich-lead` ran ~15 leads then died with `shutdown`).
- **Background mode**: `EdgeRuntime.waitUntil` was accepted (log line `[bulk-enrich] Starting background run for 150 leads`) but no `enrich-lead` invocations followed — Supabase edge runtime appears to terminate the worker as soon as the request response is sent, despite `waitUntil`.

**Recommended next step (next session):** run enrichment in batches of ~10 leads via repeated synchronous calls from the client (Pipeline header dropdown), instead of relying on a single long-running edge function. Each batch fits under the proxy timeout. Estimated 15 batches × 30s = ~8 minutes total wall clock to enrich all 150 leads at ~$3 OpenAI cost.

Alternatively: schedule the bulk-enrich on a cron (every 5 min, processing 10 leads each tick) until `firm_aum IS NULL` count hits zero.
